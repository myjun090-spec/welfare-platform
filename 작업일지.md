### 작업 2: 4개 모듈 UI 구현 (캡쳐본 기반)

#### 모듈 1 - 복지자원 통합 검색 (캡쳐 2 기반)
- **5단계 상담 프로세스** UI 구현
  - 1단계: 이용자 초기상담 (이름, 연령, 욕구, 소득수준)
  - 2단계: 검색 조건 입력 (연령대, 소득수준, 생애주기, 지원유형)
  - 3단계: 통합 검색 결과 (조건 기반 필터링)
  - 4단계: 자원 선정·연계 (담당자 배정, 메모)
  - 5단계: 모니터링·기록 (연계 상태 추적, 재사정)
- **데이터 입력 계층**: 정부·공공자원, 지역자원, 기관 프로그램 분류

#### 모듈 2 - 자원봉사자 관리 자동화 (캡쳐 1 기반)
- **3단계 자동화 흐름** 시각화
  - 1단계: 폴더 세팅 (컨텍스트, 데이터, 양식 템플릿)
  - 2단계: 코워크 실행 (실시간 문서·보고서 자동 생성)
  - 3단계: 스케줄 자동화 (매월 자동 실행, 메일 알림)
- 봉사자 등록/관리 테이블
- 보고서 생성 기능

#### 모듈 3 - 논문/연구 관리 (캡쳐 3 기반)
- **연구팀 역할 구조** 시각화
  - 연구자 → Coordinator → Scout, Analyst, Writer, Critic, Scribe
- **4라운드 Conference 구조**
  - Round 1: 주제/범위 확정
  - Round 2: 문헌 검토 토론
  - Round 3: 초안 작성
  - Round 4: 수정 후 재검토
- 연구 프로젝트 목록 및 진행률 표시

#### 모듈 4 - 팀 협업 관리 (캡쳐 4 기반)
- **팀 협업 구조** 시각화
  - 나(사람) → 팀 리더(메인 에이전트) → 작업자 A/B, 기록자 C
- 3개 탭: 팀 구조 / 결과물 / 활동 로그
- 결과물 유형: 완성된 코드, 분석 보고서, 회의록/일지

#### 대시보드
- 통계 카드 (복지자원 수, 봉사자 수, 진행중 연구, 자원 연계)
- 최근 자원 연계 목록
- 진행중 연구 프로젝트 진행률

### 작업 3: DB 스키마 설계 및 구축
- **Prisma + SQLite** 설정
- 전체 DB 모델 설계 (15개 테이블):
  - `User` - 사용자 (admin/socialworker/volunteer 역할)
  - `Resource` - 복지자원 (정부·지역·기관 분류)
  - `Client` - 이용자 (상담 대상)
  - `Consultation` - 상담 기록 (5단계 프로세스)
  - `Linkage` - 자원 연계 이력
  - `Volunteer` - 봉사자 프로필
  - `VolunteerActivity` - 봉사 활동 기록
  - `VolunteerReport` - 봉사 보고서
  - `ResearchProject` - 연구 프로젝트
  - `ResearchMember` - 연구 참여자 (역할 배정)
  - `ResearchRound` - 연구 라운드 (1~4)
  - `ResearchFile` - 연구 파일
  - `Team` - 팀
  - `TeamMember`, `TeamTask`, `TeamOutput` - 팀 협업
  - `ActivityLog` - 전체 활동 로그
- 마이그레이션 완료

### 작업 5: API 라우트 구현 (28개 엔드포인트)

| 모듈 | 엔드포인트 | 기능 |
|------|-----------|------|
| 인증 | `POST /api/auth/register` | 회원가입 |
| 인증 | `/api/auth/[...nextauth]` | 로그인/세션 |
| 복지자원 | `GET/POST /api/resources` | 자원 목록/생성 |
| 복지자원 | `GET/PUT/DELETE /api/resources/[id]` | 자원 상세/수정/삭제 |
| 이용자 | `GET/POST /api/clients` | 이용자 목록/생성 |
| 이용자 | `GET/PUT /api/clients/[id]` | 이용자 상세/수정 |
| 상담 | `GET/POST /api/consultations` | 상담 목록/생성 |
| 상담 | `PUT /api/consultations/[id]` | 상담 업데이트 |
| 연계 | `GET/POST /api/linkages` | 연계 목록/생성 |
| 연계 | `PUT /api/linkages/[id]` | 연계 상태 변경 |
| 봉사자 | `GET/POST /api/volunteers` | 봉사자 목록/등록 |
| 봉사자 | `GET/PUT /api/volunteers/[id]` | 봉사자 상세/수정 |
| 봉사자 | `GET/POST /api/volunteers/[id]/activities` | 활동 기록 |
| 봉사자 | `GET/POST /api/volunteers/[id]/reports` | 보고서 |
| 연구 | `GET/POST /api/research` | 프로젝트 목록/생성 |
| 연구 | `GET/PUT /api/research/[id]` | 프로젝트 상세/수정 |
| 연구 | `GET/POST /api/research/[id]/members` | 멤버 관리 |
| 연구 | `PUT /api/research/[id]/rounds/[roundId]` | 라운드 업데이트 |
| 팀 | `GET/POST /api/teams` | 팀 목록/생성 |
| 팀 | `GET/PUT /api/teams/[id]` | 팀 상세/수정 |
| 팀 | `GET/POST /api/teams/[id]/members` | 멤버 관리 |
| 팀 | `GET/POST /api/teams/[id]/tasks` | 작업 관리 |
| 팀 | `PUT /api/teams/[id]/tasks/[taskId]` | 작업 업데이트 |
| 팀 | `GET/POST /api/teams/[id]/outputs` | 결과물 관리 |
| 공통 | `GET/POST /api/activity-logs` | 활동 로그 |
| 시드 | `GET /api/seed` | 샘플 데이터 생성 |
